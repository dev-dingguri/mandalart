# SEO Phase 1 — 최소 SEO 인프라 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** SEO 최적화를 위한 랜딩 페이지와 가이드 페이지를 추가하고, 기술적 SEO 인프라(robots.txt, sitemap, 구조화 데이터)를 구축한다.

**Architecture:**
- 기존 `/{lang}/` → 콘텐츠 랜딩 페이지로 변경, 기존 도구는 `/{lang}/app`으로 이동
- 재방문자(localStorage `has_used_tool` 플래그)는 `/{lang}/` 접근 시 `/{lang}/app`으로 자동 리다이렉트
- react-helmet-async로 페이지별 `<head>` 메타 태그 관리
- 빌드 시 프리렌더링으로 콘텐츠 페이지 정적 HTML 생성

**Tech Stack:** React 19, TypeScript 5, Tailwind CSS v4, shadcn/ui, react-helmet-async, Vitest

**콘텐츠 참조:** `docs/seo-content-plan.md` — 각 페이지의 상세 콘텐츠 구성과 타겟 키워드 정의

## 설계 결정 사항

### 다국어 랜딩 페이지 동작

Phase 1에서 콘텐츠 페이지(랜딩/가이드)는 한국어 하드코딩이다. 비한국어 사용자(`/en`, `/ja`, `/zh-CN`)가 `/{lang}/`에 접근하면:
- 랜딩 페이지를 보여주되, 콘텐츠는 한국어로 표시된다.
- 추후 다국어 콘텐츠가 준비되면 언어별 분기를 추가한다.
- `has_used_tool` 플래그가 있으면 언어와 무관하게 `/{lang}/app`으로 리다이렉트한다.

### hreflang 처리

Phase 1에서는 `index.html`의 기존 hreflang 태그를 유지한다. hreflang은 `/{lang}/` 루트를 가리키고 있으며, 라우팅 변경 후에도 이 URL이 각 언어의 대표 페이지이므로 수정 불필요. `/guide` 등 새 페이지의 hreflang은 다국어 콘텐츠 추가 시 함께 구현한다.

```tsx
import { PATH_OSS, PATH_APP, PATH_GUIDE } from '@/constants';

// lazy imports 추가
const LandingPage = lazy(() => import('@/components/LandingPage'));
const GuidePage = lazy(() => import('@/components/GuidePage'));

// 기존 <div className="h-dvh"> 제거, 라우트별로 높이 처리:
<BrowserRouter>
  <Routes>
    <Route path={`/${lang}`} element={
      <Suspense fallback={null}><LandingPage /></Suspense>
    } />
    <Route path={`/${lang}${PATH_APP}`} element={
      <div className="h-dvh"><MainPage /></div>
    } />
    <Route path={`/${lang}${PATH_GUIDE}`} element={
      <Suspense fallback={null}><GuidePage /></Suspense>
    } />
    <Route path={`/${lang}${PATH_OSS}`} element={
      <Suspense fallback={null}><OpenSourceLicensesPage /></Suspense>
    } />
    <Route path="*" element={<Navigate to={`/${lang}`} />} />
  </Routes>
</BrowserRouter>
```

> **Why h-dvh 분기:** 만다라트 도구(`/app`)는 고정 뷰포트 앱이라 `h-dvh`가 필요하지만, 랜딩/가이드 페이지는 스크롤 가능한 긴 콘텐츠이므로 `h-dvh`로 감싸면 스크롤이 불가능하다.

import { describe, it, expect, beforeEach } from 'vitest';
import { STORAGE_KEY_HAS_USED_TOOL } from '@/constants';

// 플래그 설정 로직을 검증하는 헬퍼 (실제 로직과 동일)
const markToolUsed = () => {
  if (!localStorage.getItem(STORAGE_KEY_HAS_USED_TOOL)) {
    localStorage.setItem(STORAGE_KEY_HAS_USED_TOOL, 'true');
  }
};

describe('has_used_tool flag', () => {
  beforeEach(() => {
    localStorage.clear();
  });

  it('should not exist for first-time visitors', () => {
    expect(localStorage.getItem(STORAGE_KEY_HAS_USED_TOOL)).toBeNull();
  });

  it('should be set after markToolUsed is called', () => {
    markToolUsed();
    expect(localStorage.getItem(STORAGE_KEY_HAS_USED_TOOL)).toBe('true');
  });

  it('should not overwrite existing flag', () => {
    localStorage.setItem(STORAGE_KEY_HAS_USED_TOOL, 'true');
    markToolUsed(); // 중복 호출
    expect(localStorage.getItem(STORAGE_KEY_HAS_USED_TOOL)).toBe('true');
  });
});

- [ ] **Step 2: App.tsx에 프리렌더링 완료 이벤트 추가**

```tsx
useEffect(() => {
  // 프리렌더러에 렌더링 완료를 알림

}, []);
```

        renderAfterDocumentEvent: 'render-event',
