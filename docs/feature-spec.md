# Mandalart — 기능 명세서

> 소스코드 기반으로 역추출한 기능 명세서입니다. Vite 6 마이그레이션 및 컴포넌트 리팩토링 완료 후 현행화됨 (2026-03-25).

---

## 1. 제품 개요

| 항목 | 내용 |
|------|------|
| **제품명** | Mandalart |
| **목적** | 하나의 목표를 81칸(9×9) 격자로 분해하여 구체적인 실행 계획을 수립하는 목표 설정 도구 |
| **플랫폼** | 웹 (모바일 최적화 포함) |
| **지원 언어** | 한국어, English, 日本語, 中文(简体) |

---

## 2. 데이터 모델

```
만다라트 1개 = 메타(제목) + 토픽 트리
토픽 트리 = 1개 루트 → 8개 하위 목표 → 각 8개 실행 항목 = 총 73개 노드
```

**격자와 노드의 관계**: 9×9 격자는 81칸이지만 데이터 노드는 73개이다. 차이(8칸)는 **미러 셀**에 해당한다. 중앙 3×3 그리드의 8개 주변 셀(레벨 1 하위 목표)이 각 주변 그리드의 중앙 셀에 중복 표시된다. 미러 셀은 시각적으로 강조(`isAccented`) 처리되며, 편집 시 원본 노드와 동일한 데이터를 수정한다.

| 제약 | 값 |
|------|-----|
| 최대 만다라트 수 | 20개 |
| 제목 최대 길이 | 50자 |
| 토픽 텍스트 최대 길이 | 50자 |
| 격자 크기 | 9×9 (3×3 그리드 × 9), 이 중 8칸은 미러 셀 |

## 3. 사용자 모드

### 3.1 게스트 모드 (미로그인)

| 항목 | 내용 |
|------|------|
| **데이터 저장** | 브라우저 localStorage |
| **만다라트 개수** | 1개 (임시) |
| **제한** | 새 만다라트 생성 불가, 다른 기기 동기화 불가 |
| **라벨** | 제목 위에 "(이 기기에만)" 표시 |
| **삭제** | "삭제" 대신 "초기화" 제공 |

### 3.2 인증 모드 (로그인)

| 항목 | 내용 |
|------|------|
| **데이터 저장** | Firebase Realtime Database |
| **만다라트 개수** | 최대 20개 |
| **동기화** | 실시간 양방향 동기화 (Firebase `onValue` 구독) |
| **로그인 방식** | Google OAuth (팝업) |

### 3.3 게스트 → 인증 전환 (데이터 마이그레이션)

- 로그인 시 게스트 데이터를 Firebase로 자동 업로드
- **첫 번째 만다라트만** 마이그레이션됨 (Map 삽입 순서 기준)
- `sessionStorage` 플래그(`shouldUploadTemp`)로 중복 업로드 방지. 단, 진정한 중복 방지는 `uploadTemp` 내부의 localStorage 유효성 체크에 의존
- 업로드 시도 **전에** 플래그를 `false`로 설정하므로, 실패 시 **같은 세션에서 재시도 불가** (새 탭에서는 `sessionStorage` fallback에 의해 재시도됨)
- 마이그레이션 성공 시 사용자 피드백 없음 (분석 이벤트만 기록)
- 마이그레이션 실패 시 AlertDialog로 에러 표시하나, 재시도 안내 없음

## 4. 핵심 기능

### 4.1 만다라트 CRUD

| 기능 | 진입점 | 동작 | 비고 |
|------|--------|------|------|
| **생성** | 빈 화면 중앙 버튼 / 좌측 드로어 하단 버튼 | 빈 만다라트 생성 후 자동 선택 | 게스트 불가, 최대 20개. 빈 화면은 인증 모드에서 만다라트 0개일 때만 노출 |
| **선택** | 좌측 드로어 목록 클릭 | 해당 만다라트로 전환 | 전환 중 빈 화면 방지 처리 |
| **삭제** | 좌측 드로어 → 항목 "..." 메뉴 → 삭제 | 확인 다이얼로그 후 삭제 | 메타+트리 원자적 삭제. 현재 보고 있는 만다라트 삭제 시 **마지막 항목(Map 삽입 순서 기준)**으로 자동 전환. 전부 삭제 시 빈 화면 표시 |
| **초기화** | 좌측 드로어 → 항목 "..." 메뉴 → 초기화 | 확인 다이얼로그 후 내용 비움 | 구조 유지, 텍스트만 제거. 게스트 임시 만다라트 전용 |
| **이름 변경** | 헤더 제목 영역 클릭 | 인라인 텍스트 편집 (`useTitleEdit` 훅) | 최대 50자. Enter로 저장, Escape/blur로 취소. 만다라트 전환 시 편집 자동 취소 |

### 4.2 토픽 편집

반응형 입력 UI — 데스크탑과 모바일에서 다른 입력 경로를 사용한다.

**데스크탑** (`min-width: 48rem`):
- 셀 클릭 → `PopoverCellInput` (Radix Popover 기반, 셀 옆에 표시)
- 좌우 이동 버튼(←→) + 확인 버튼(✓) + 글자 수 카운터
- 셀 위치 표시 (예: "3A")

**모바일**:
- 셀 클릭 → `BottomInputBar` (화면 하단 고정)
- `visualViewport` API로 소프트 키보드 위에 위치 조정 (iOS Safari 대응)
- 동일한 레이아웃 (←→, ✓, 글자 수 카운터)

**공통 동작** (`useCellInput` 훅):
- 최대 50자, 초과 시 카운터 강조 + 확인 버튼 비활성화
- 키보드: Enter/Tab → 다음 셀, Shift+Tab → 이전 셀, ↑↓ → 위/아래 셀, Escape → 닫기
- `cellKey` 변경 시 텍스트 리셋 + 입력 포커스 (셀 간 이동 시 자연스러운 전환)
- 저장 즉시 Firebase 동기화 (인증 모드), toast 에러 알림 (저장 실패 시)

### 4.3 뷰 모드

| 모드 | 설명 | 전환 방법 |
|------|------|-----------|
| **전체 보기 (All View)** | 9×9 전체 격자 표시 | 기본값, 축소 아이콘 클릭 |
| **확대 보기 (Focus View)** | 선택한 3×3 그리드 확대 | 확대 아이콘 클릭 |

**뷰 모드 세부 동작:**

- Focus View 전환 시 **항상 중앙 그리드(인덱스 4)** 에서 시작. All View에서 보던 위치와 무관
- All View ↔ Focus View 전환 시 `isAllView` 상태는 `MandalartView` 내부 상태이므로, **만다라트를 전환해도 뷰 모드가 유지**됨
- Focus View에서 **비포커스 그리드 셀 클릭** 시 편집이 아닌 해당 그리드로 **포커스 이동**. 편집하려면 포커스 이동 후 다시 클릭 필요
- **키보드 방향키**로 Focus View 그리드 간 이동 가능 (상하좌우). 컨테이너 진입 시 자동 포커스

### 4.4 스와이프 네비게이션 (Focus View 전용, 터치만)

- 터치 스와이프로 인접 그리드 이동
- **상하좌우 + 대각선** 방향 지원 (수직·수평 독립 계산, 경계에서 부분 이동 가능)
- 플릭 제스처 감지: 500ms 이내 빠른 스와이프 시 속도 가중치 적용 (최대 5배)
- 스와이프 임계값: 컨테이너 너비 × 35%
- 이동 실패 시 원래 위치로 smooth 스크롤 복원
- **키보드 방향키**: 상하좌우 이동 지원 (대각선은 터치만). Focus View 진입 시 컨테이너에 자동 포커스되어 즉시 방향키 사용 가능

## 7. UI 구조

### 7.1 헤더

```
[메뉴 아이콘] ─── [Mandalart 제목] ─── [아바타/로그인] [설정("...")]
```

- **아바타**: 사용자 이름 첫 글자 (원형 배지), 클릭 시 이름/이메일/로그아웃 드롭다운
- **미로그인 시**: "로그인" 텍스트 버튼

### 7.2 좌측 드로어 — 만다라트 목록

- Radix Dialog 기반 Sheet 컴포넌트 (왼쪽에서 슬라이드)
- 만다라트 목록 (최신순)
- 현재 선택 항목 강조
- 각 항목 "..." 메뉴(`MandalartListItemMenu`): 삭제 또는 초기화. 삭제·초기화 구분은 사용자 인증 상태가 아닌 **만다라트 ID가 임시 ID(`TMP_MANDALART_ID`)인지 여부**로 결정
- 하단 "새 만다라트" 버튼 (게스트 모드에서도 표시됨 — 클릭 시 로그인 필요 에러)

### 7.3 우측 드로어 — 설정

- Radix Dialog 기반 Sheet 컴포넌트 (오른쪽에서 슬라이드)
- 테마, 언어, OSS 라이선스, 버전, 연락처

### 7.4 다이얼로그

| 다이얼로그 | 용도 | 특징 |
|------------|------|------|
| **ConfirmDialog** | 삭제/초기화 확인 | Radix AlertDialog 기반, 커스텀 확인 텍스트, Cancel/Confirm 2버튼 |
| **AlertDialog** | 에러 알림 | Radix AlertDialog 기반, ESC/외부 클릭 차단, OK 1버튼 |
| **SignInDialog** | 로그인 유도 | Radix Dialog 기반, Google 로그인 버튼 + 안내 문구 |

### 7.5 인라인 입력 UI

토픽 편집과 제목 편집은 다이얼로그가 아닌 인라인 방식으로 동작한다.

| 컴포넌트 | 용도 | 조건 |
|----------|------|------|
| **PopoverCellInput** | 토픽 셀 편집 | 데스크탑 (`min-width: 48rem`) — Radix Popover로 셀 옆에 표시 |
| **BottomInputBar** | 토픽 셀 편집 | 모바일 — 화면 하단 고정, `visualViewport` API로 키보드 대응 |
| **인라인 제목 편집** | 만다라트 이름 변경 | 헤더 제목 클릭 → `useTitleEdit` 훅으로 인라인 편집 |

## 8. 콘텐츠 페이지

도구 페이지(`/{lang}/app`)와 별도로, SEO 최적화된 콘텐츠 페이지를 제공한다. 콘텐츠 페이지는 빌드 타임에 프리렌더링되어 정적 HTML로 서빙된다.

### 8.1 랜딩 페이지 (`/{lang}`)

| 항목 | 내용 |
|------|------|
| **목적** | 신규 방문자에게 만다라트를 소개하고 도구 사용을 유도 |
| **재방문자 리다이렉트** | `localStorage`의 `has_used_tool` 플래그가 `true`이면 `/{lang}/app`으로 자동 리다이렉트 |
| **플래그 설정 시점** | 토픽 또는 메타 데이터를 **처음 저장**할 때 (게스트·인증 모드 공통) |

**페이지 구성 (8개 섹션):**

| 순서 | 섹션 | 내용 |
|------|------|------|
| 1 | Hero | 헤드카피 + "지금 시작하기" CTA 버튼 |
| 2 | 3단계 사용법 | 중심 목표 → 8개 핵심 요소 → 64개 실행 계획 |
| 3 | Before/After | "막연한 목표" vs "81칸으로 구체화된 계획" 비교 |
| 4 | 활용 사례 | 취업 준비, 다이어트, 자기계발, 사업 계획 카드 |
| 5 | 소셜 프루프 | 오타니 쇼헤이 사례 + 가이드 페이지 링크 |
| 6 | FAQ | `<details>`/`<summary>` 아코디언, FAQ JSON-LD 구조화 데이터 포함 |
| 7 | Final CTA | "무료로 시작하기" 버튼 |

### 8.2 가이드 페이지 (`/{lang}/guide`)

| 항목 | 내용 |
|------|------|
| **목적** | "만다라트란?" 검색 의도에 대응하는 정보형 콘텐츠 |
| **내비게이션** | 상단 sticky 바 — "홈으로" 링크 |

**페이지 구성 (7개 섹션):**

| 순서 | 섹션 | 내용 |
|------|------|------|
| 1 | H1 + 부제 | "만다라트(Mandalart)란?" |
| 2 | 기원과 역사 | 이마이즈미 히로아키, 만다라에서 영감, 이름의 어원 |
| 3 | 구조 — 81칸의 비밀 | 1칸(핵심 목표) + 8칸(핵심 요소) + 64칸(세부 행동) 구조 카드 |
| 4 | 작성법 5단계 | SMART 목표 설정 → 핵심 요소 도출 → 세부 행동 → 우선순위 → 실행/리뷰 |
| 5 | 비교표 | 만다라트 vs 마인드맵/OKR/비전보드/버킷리스트 (구체성·전체 조망·난이도 별점) |
| 6 | 오타니 쇼헤이 사례 | 중심 목표·8개 핵심 요소·'운' 세부 행동 상세 |
| 7 | CTA | "시작하기" 버튼 |

### 8.3 SEO 인프라

| 항목 | 내용 |
|------|------|
| **메타 태그** | `SEOHead` 컴포넌트 — title, description, canonical, Open Graph (`react-helmet-async`) |
| **프리렌더링** | `@prerenderer/rollup-plugin` — `/ko`, `/ko/guide`를 빌드 시 정적 HTML로 생성 |
| **robots.txt** | `public/robots.txt` — 전체 허용 + sitemap 참조 |
| **sitemap.xml** | `public/sitemap.xml` — 모든 공개 경로 포함, 콘텐츠 페이지 우선순위 높음 |
| **구조화 데이터** | 랜딩 페이지 FAQ 섹션에 `FAQPage` JSON-LD 스키마 |
| **스크롤 초기화** | `ScrollToTop` 컴포넌트 — 라우트 변경 시 `window.scrollTo(0, 0)` 호출 |

### 8.4 레이아웃 전략

| 페이지 유형 | 높이 전략 | 스크롤 |
|-------------|-----------|--------|
| 콘텐츠 페이지 (랜딩, 가이드) | `min-h-dvh` | document 스크롤 (긴 콘텐츠) |
| 도구 페이지 (메인, OSS) | `h-dvh` 래퍼 | 내부 overflow 스크롤 (고정 뷰포트) |

`body`에 `overscroll-behavior: none`을 설정하여 모바일 pull-to-refresh를 방지하되, document 스크롤은 허용한다.

## 9. 에러 메시지

### 9.1 정의된 에러 메시지 (i18n 리소스 기준)

| 상황 | i18n 키 | 메시지 (ko) |
|------|---------|-------------|
| 로그인 실패 | `auth.errors.signIn` | "로그인할 수 없습니다. 잠시 후 다시 시도해주세요." |
| 생성 실패 (미로그인) | `mandalart.errors.create.signInRequired` | "만다라트를 추가할 수 없습니다. 😥\n추가하려면 로그인이 필요합니다." |
| 생성 실패 (한도 초과) | `mandalart.errors.create.maxUploaded` | "만다라트를 추가할 수 없습니다. 😥\n최대 {{maxSize}}개까지 저장할 수 있습니다.\n기존 만다라트를 삭제한 후 다시 시도해 주세요." |
| 생성 실패 (기타) | `mandalart.errors.create.default` | "만다라트를 추가할 수 없습니다. 😥" |
| 동기화 에러 | `mandalart.errors.sync.default` | "로그인 세션이 만료되었습니다.\n다시 로그인해 주세요." |

### 9.2 에러 처리 현황

| 상황 | 현재 동작 |
|------|-----------|
| 토픽/제목 저장 실패 (네트워크 오류 등) | `toast.error(t('mandalart.errors.save.default'))` — 비차단 토스트 알림 (`sonner`) |
| 삭제/초기화 실패 | Firebase SDK raw 에러 메시지 그대로 노출 (i18n 미적용) |
| 동기화 에러 후 | AlertDialog 확인 → 페이지 새로고침 (`window.location.reload`) |
| 마이그레이션 한도 초과 | 전용 메시지(`uploadTemp.maxUploaded`)가 i18n에 존재하나 **코드에서 미사용** — 일반 생성 에러가 표시됨 |

> **참고**: `mandalart.errors.sync.default`는 "세션 만료"뿐 아니라 네트워크 오류, 권한 거부 등 **모든 Firebase `onValue` 구독 에러**에 동일하게 표시된다.

## 12. 알려진 제한사항

- **저장 성공 피드백 없음**: Firebase 실시간 구독으로 UI가 즉시 반영되므로 별도 토스트 없음
- **게스트 마이그레이션**: 첫 번째 만다라트만 이관, 나머지는 유실
- **마이그레이션 실패 시 재시도 불가**: `shouldUploadTemp` 플래그가 시도 전에 `false`로 설정되므로 같은 세션에서 재시도 경로 없음 (새 탭 열기로 우회 가능)
- **동기화 에러 후 강제 새로고침**: Firebase `onValue` 에러 발생 시 구독 자동 취소. AlertDialog 확인 후 `window.location.reload()`로 페이지를 새로고침하여 복구
- **게스트 다중 탭 충돌**: 게스트 모드에서 여러 탭 동시 편집 시 localStorage 쓰기가 서로 덮어씀. `storage` 이벤트 리스너 없음
- **오프라인 지원 없음**: 네트워크 연결 필수
