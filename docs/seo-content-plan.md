# SEO 콘텐츠 페이지 기획안

## 전체 전략: 정보형 콘텐츠 → 도구 전환 퍼널

현재 사이트의 가장 큰 SEO 문제는 **크롤링할 텍스트 콘텐츠가 없다**는 것이다. 검색엔진은 SPA의 도구 UI를 콘텐츠로 인식하지 못한다. 아래 페이지들은 검색 의도(Search Intent)별로 설계했다.

---

## 1. 랜딩 페이지 (메인) — `/{lang}/`

> 검색 의도: 브랜드 / 네비게이션

현재 바로 도구로 진입하는 구조를, **히어로 섹션 + 가치 제안 + CTA**로 전환한다.

### 헤드라인 후보

**메인 헤드라인**
- "꿈을 현실로 바꾸는 81칸의 마법, 만다라트(Mandalart)"
- "단 하나의 목표에서 시작하는 81가지 실행 전략"

**서브 태그라인**
- 행동 강조: "막연한 결심을 구체적인 행동으로. 만다라트로 당신의 실행력을 200% 높이세요."
- 쉬운 사용성: "큰 꿈을 작은 습관으로 쪼개는 가장 스마트한 방법, 지금 바로 시작하세요."
- 본질 강조: "생각의 확장이 목표의 달성으로 이어지는 순간, 만다라트가 함께합니다."

### 섹션 구성

| 섹션 | 콘텐츠 | 목적 |
|------|--------|------|
| **Hero** | 메인 헤드라인 + 서브 태그라인 + "지금 시작하기" CTA | 즉시 가치 전달 |
| **3-Step 사용법** | ① 중심 목표 입력 → ② 8개 핵심 요소 도출 → ③ 64개 실행 계획 완성 | 사용 허들 낮추기 |
| **비포/애프터** | "막연한 목표" vs "81칸으로 구체화된 계획" 비교 시각화 | 가치 증명 |
| **사용 사례 카드** | 취업 준비 / 다이어트 / 자기계발 / 사업 계획 등 4-6개 | 공감 유도 |
| **소셜 프루프** | 사용자 수, 후기, "오타니 쇼헤이도 사용한 방법" 언급 | 신뢰 구축 |
| **FAQ (구조화 데이터)** | 만다라트란? / 무료인가요? / 데이터 저장은? | FAQ 리치 스니펫 |
| **최종 CTA** | "지금 바로 나만의 만다라트를 만들어보세요" | 전환 |

### 타겟 키워드
- 주: `만다라트`, `만다라트 계획표`, `mandalart`
- 부: `목표 설정 도구`, `만다라트 만들기`, `81칸 계획표`

## 2. "만다라트란?" 가이드 페이지 — `/{lang}/guide`

> 검색 의도: 정보 탐색 (Informational)
>
> **가장 핵심적인 SEO 페이지**. "만다라트"를 검색하는 사용자의 대부분은 이 개념을 처음 접했거나 자세히 알고 싶은 사람이다.

### 콘텐츠 구성

```
H1: 만다라트(Mandalart)란? — 목표 달성을 위한 최고의 사고 확장 도구

H2: 만다라트의 기원과 역사
  - 1987년 디자이너 이마이즈미 히로아키가 개발
  - 불교 만다라에서 영감을 받은 구조
  - "목표 달성(Manda) + 기술(Art)"의 합성어

H2: 만다라트의 구조 — 81칸의 비밀
  - 3×3 중심 매트릭스 설명 (인터랙티브 다이어그램)
  - 중심 목표 → 8개 핵심 요소 → 64개 세부 행동
  - 왜 "81칸"인지에 대한 논리적 설명

H2: 만다라트 작성법 — 5단계 가이드
  ① 중심 목표 설정 (SMART 원칙 적용)
  ② 8개 핵심 요소 도출 (브레인스토밍 팁)
  ③ 각 요소의 세부 행동 8개씩 작성
  ④ 우선순위 정하기
  ⑤ 실행 및 리뷰

H2: 만다라트 vs 다른 목표 설정 방법
  - 만다라트 vs 마인드맵
  - 만다라트 vs OKR
  - 만다라트 vs 비전보드
  - 만다라트 vs 버킷리스트
  (표 형식 비교)

H2: 오타니 쇼헤이의 만다라트
  - 고교 시절 작성한 만다라트 사례
  - "드래프트 1순위 8구단 지명" 목표
  - 실제 달성한 결과와의 비교 분석

H2: 만다라트 활용 분야
  - 개인 목표 (건강, 자기계발, 재테크)
  - 학업 (수능, 자격증, 진학)
  - 비즈니스 (사업 계획, 프로젝트 관리)
  - 팀 워크숍 (조직 비전 수립)

[CTA] 이론은 충분합니다. 지금 바로 만다라트를 작성해보세요 →
```

### 타겟 키워드
- 주: `만다라트란`, `만다라트 뜻`, `만다라트 작성법`
- 부: `오타니 만다라트`, `만다라트 사용법`, `81칸 계획표 만드는 법`
- 롱테일: `만다라트 작성 방법`, `만다라트 예시`, `목표 설정 방법`

## 3. 만다라트 예시/템플릿 페이지 — `/{lang}/examples`

> 검색 의도: 정보 탐색 + 상업적 조사
>
> "만다라트 예시", "만다라트 양식"을 검색하는 사용자는 **직접 만들 의향이 높은** 고가치 트래픽이다.

### 콘텐츠 구성

```
H1: 만다라트 예시 모음 — 바로 따라 할 수 있는 템플릿

H2: 자기계발 만다라트
  - 예시: "2026년 최고의 나 되기"
  - 완성된 81칸 시각적 표시 (인터랙티브)
  - [이 템플릿으로 시작하기] CTA

H2: 취업 준비 만다라트
  - 예시: "상반기 대기업 합격"
  - 핵심 요소: 직무역량, 자소서, 면접, 체력관리 등

H2: 다이어트/건강 만다라트
  - 예시: "6개월 안에 10kg 감량"

H2: 학업/수능 만다라트
  - 예시: "수능 상위 3% 달성"

H2: 사업 계획 만다라트
  - 예시: "1인 창업 6개월 안에 첫 매출"

H2: 신년 계획 만다라트
  - 예시: "2026년 버킷리스트 달성"

H2: 팀/조직 만다라트
  - 예시: "Q1 프로젝트 성공적 런칭"

각 예시마다:
  - 완성된 만다라트 이미지/인터랙티브 뷰
  - 작성 과정 해설
  - "이 템플릿으로 바로 시작하기" CTA 버튼
```

### 타겟 키워드
- 주: `만다라트 예시`, `만다라트 템플릿`, `만다라트 양식`
- 부: `만다라트 다이어트`, `만다라트 취업`, `만다라트 자기계발`
- 롱테일: `만다라트 계획표 예시`, `2026 만다라트`, `새해 목표 만다라트`

## 4. 블로그/팁 섹션 — `/{lang}/blog`

> 검색 의도: 정보 탐색 (롱테일 키워드 포획)

개별 글은 하위 경로(`/blog/{slug}`)로 구성한다. 초기 5-7개 글로 시작하고 점진적으로 확장한다.

### 초기 콘텐츠 로드맵

| # | 제목 (H1) | 타겟 키워드 | 우선순위 |
|---|-----------|------------|---------|
| 1 | 오타니 쇼헤이가 고교 시절 작성한 만다라트 분석 | `오타니 만다라트`, `오타니 목표` | ★★★ |
| 2 | 만다라트 작성할 때 흔히 하는 5가지 실수 | `만다라트 작성법`, `만다라트 팁` | ★★★ |
| 3 | 새해 목표, 작심삼일로 끝나지 않는 방법 | `새해 목표 세우기`, `작심삼일 극복` | ★★☆ |
| 4 | 만다라트 vs 마인드맵, 어떤 게 나에게 맞을까? | `만다라트 마인드맵 차이` | ★★☆ |
| 5 | 목표 설정이 어려운 사람을 위한 만다라트 시작 가이드 | `목표 설정 방법`, `목표 찾기` | ★★☆ |
| 6 | 만다라트로 다이어트 계획 세우는 법 | `다이어트 계획`, `만다라트 다이어트` | ★☆☆ |
| 7 | 수능/자격증 합격을 위한 만다라트 활용법 | `수능 계획표`, `만다라트 공부` | ★☆☆ |

## 5. FAQ 페이지 — `/{lang}/faq`

> 검색 의도: 정보 탐색 (음성 검색 / People Also Ask 대응)

### 필수 질문 목록

- 만다라트란 무엇인가요?
- 만다라트는 누가 만들었나요?
- 만다라트와 마인드맵의 차이점은?
- 만다라트를 작성하는 데 얼마나 걸리나요?
- 만다라트를 팀으로 함께 작성할 수 있나요?
- 만다라트 계획표는 무료인가요?
- 작성한 만다라트를 저장할 수 있나요?
- 만다라트를 인쇄/다운로드할 수 있나요?
- 만다라트는 몇 칸인가요?
- 오타니 쇼헤이의 만다라트는 어떻게 생겼나요?

> **구조화 데이터(FAQ Schema)** 적용 필수 — Google 검색 결과에서 FAQ 리치 스니펫으로 노출된다.

---

## URL 구조 및 라우팅 설계

```
/{lang}/                 → 랜딩 페이지 (리뉴얼)
/{lang}/app              → 기존 도구 (만다라트 작성기)
/{lang}/guide            → 만다라트 가이드
/{lang}/examples         → 예시/템플릿
/{lang}/blog             → 블로그 목록
/{lang}/blog/{slug}      → 블로그 글
/{lang}/faq              → FAQ (선택)
```

## 기술적 SEO 체크리스트

| 항목 | 현재 | 개선 |
|------|------|------|
| `robots.txt` | 없음 | 생성 필요 |
| `sitemap.xml` | 오래된 날짜 | 새 페이지 추가 + 날짜 갱신 |
| 구조화 데이터 | 없음 | FAQ Schema, HowTo Schema, BreadcrumbList |
| canonical URL | 없음 | 각 페이지에 추가 |
| Open Graph | 기본만 | 페이지별 고유 OG 이미지/설명 |
| 내부 링크 | 없음 | 콘텐츠 페이지 간 상호 링크 |
| 페이지 속도 | SPA 전체 로딩 | 콘텐츠 페이지는 SSG 또는 경량 렌더링 고려 |

## 콘텐츠 우선순위 로드맵

### Phase 1 (즉시) — 최소 SEO 인프라
1. **랜딩 페이지 리뉴얼** — 히어로 + 3단계 + CTA
2. **가이드 페이지** (`/guide`) — 가장 많은 검색 트래픽 유입원
3. `robots.txt` + `sitemap.xml` 갱신
4. FAQ 구조화 데이터 (랜딩 하단)

### Phase 2 (1-2주 내) — 전환 극대화
5. **예시/템플릿 페이지** (`/examples`) — 도구 사용 전환율 가장 높은 페이지
6. 페이지별 OG 이미지 제작
7. 내부 링크 구조 구축

### Phase 3 (지속적) — 롱테일 트래픽
8. **블로그 첫 3개 글** — 오타니, 작성 실수, 새해 목표
9. 추가 블로그 글 지속 발행
10. 사용자 사례 수집 및 콘텐츠화

## 각 페이지별 핵심 성과 지표 (KPI)

| 페이지 | 핵심 지표 | 목표 |
|--------|----------|------|
| 랜딩 | 도구 전환율 (CTA 클릭) | 30%+ |
| 가이드 | 평균 체류 시간 + 도구 이동률 | 3분+ / 20%+ |
| 예시 | 템플릿 시작 클릭률 | 40%+ |
| 블로그 | 오가닉 유입 + 도구 이동률 | 월 1000+ / 15%+ |
